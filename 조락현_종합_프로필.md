## 🛠️ 기술 스택

### 펌웨어 개발
- **MCU**: STM32 (G0/F4/F7/H7 시리즈) - 주력
- **RTOS**: FreeRTOS (Task 관리, Queue, Semaphore)
- **통신**: UART, SPI, I2C, Modbus RTU/TCP, Ethernet, CAN
- **제어**: BLDC 모터 제어, PID 알고리즘, 스테퍼 모터
- **센서**: ADC/DAC, 광학 센서, 온습도 센서, pH 센서
- **GUI**: TouchGFX, 간단한 UI 구현

### 하드웨어 설계
- **CAD 툴**: PADS (주 사용), OrCAD (Schematic)
- **회로 설계**: 
  - 디지털: MCU 주변 회로, 메모리 인터페이스
  - 아날로그: Op-amp 회로, ADC/DAC, 센서 신호 처리
  - 전원: SMPS (Buck/Boost), LDO 설계
  - 고전압: -1500V 고전압 PCB (최근 경험)
  - 통신: RS-485, Ethernet, Isolation 회로
- **PCB 설계**: 4~6층 PCB, 신호 무결성, 노이즈 필터링
- **검증**: 오실로스코프, 로직 분석기, 멀티미터

### 기타
- **FPGA**: 통신 인터페이스 연동 (설계는 비전문)
- **문서화**: 회로도, 매뉴얼, 디버깅 리포트 작성
- **협업**: 하드웨어-펌웨어 통합 팀 협업, 주간 설계 리뷰

## 💪 핵심 강점

### 1. 하드웨어-펌웨어 통합 디버깅 능력
- 팀에서 유일하게 회로와 펌웨어를 모두 다룸
- 문제 발생 시 하드웨어/펌웨어 원인 구분 가능
- 예시:
  - L-LPC ADC 노이즈: 디지털 신호 간섭 → 펌웨어 필터 + PCB 재설계
  - MS uA741 Op-amp: Rail-to-rail 한계 → 칩 교체로 해결
  - Psi-1000 Modbus: RS-485 방향 제어 타이밍 이슈 → MCU I2C Errata 발견

### 2. 책임감과 협업 능력
- 양쪽 관점에서 해결 방향 제시: "펌웨어에서 보완 가능", "회로 수정이 더 확실"
- 매주 설계 리뷰에서 회로도와 펌웨어 구조 함께 검토
- 의견 조율 및 일정 조율 역할 자연스럽게 수행

### 3. 빠른 분석과 다각도 문제 해결
- 새로운 기술도 빠르게 적응 (고전압 PCB 첫 시도 성공)
- 오실로스코프, 코드, 회로도 동시 분석
- 프로토 단계 임시 해결 → 차기 보드 근본 해결

## 🔍 주요 디버깅 사례

### 1. L-LPC ADC 노이즈 이슈
- **문제**: ADC 입력 노이즈 심함
- **원인**: 디지털 신호 라인이 ADC 입력에 너무 근접
- **해결**: 
  1. 펌웨어 Moving Average 필터 임시 적용
  2. 회로팀과 협업하여 PCB 재배치 설계
- **결과**: 노이즈 90% 감소, 안정적 측정

### 2. Psi-1000 Modbus 통신 간헐적 실패
- **문제**: Modbus RTU 통신 가끔 끊김
- **디버깅**: 
  - 오실로스코프로 RS-485 파형 분석
  - Transceiver 방향 제어 타이밍 확인
  - 코드 한 줄씩 검토
  - 2주간 집중 디버깅
- **해결**: MCU I2C Errata 문서 발견 → 재초기화 로직 추가
- **교훈**: 끈질긴 디버깅 + Datasheet/Errata 중요성

### 3. MS uA741 Op-amp 출력 부족
- **문제**: DAC 출력이 0~10V 필요한데 일부 범위만 출력
- **원인**: uA741이 이전 회로에서 재사용됐는데, 단일 전원에서 Rail-to-rail 불가
- **해결**: Rail-to-rail Op-amp로 교체 (핀 호환 제품)
- **결과**: 전 범위 정상 출력

### 4. SEM 고전압 PCB 스파크
- **문제**: -1500V 고전압 출력에서 스파크 발생
- **원인**: HV Power Supply와 Output 패턴이 같은 Top층에 너무 근접
- **해결**: 
  1. 즉시 Top층 패턴 제거
  2. Bottom면 와이어 연결로 프로토 완성
  3. 차기 보드: 층 분리 설계 (1000V 이상 10mm 이상 거리)
- **교훈**: UART Isolation처럼 고전압도 거리 확보 필수

### 5. MS 1차 프로토 11개 하드웨어 오류 발견
1. DAC Ref pin 캐패시터 누락
2. RS232 Isolation 다이오드 역방향
3. 커넥터 회전 문제
4. ADC 범위 포화 (9.868V vs 10V)
5. Op-amp 출력 스윙 부족
6. Differential ADC 설계 오류
7. Relay artwork A1/A2 스왑
8. ~ 11. 전압 범위 및 전원 제어 이슈
