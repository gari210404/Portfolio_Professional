# 원익로보틱스 면접 준비 자료

## 📌 회사 정보
- **회사명**: 원익로보틱스
- **포지션**: 로봇 핸드 펌웨어 엔지니어
- **주력 제품**: 20-DOF 로봇 핸드, 멀티모달 센서 융합, 실시간 제어
- **특징**: 산업용/협동 로봇, 정밀 제어 기술

## 🎯 1분 자기소개

안녕하세요. 9년간 임베디드 시스템 개발을 해온 조락현입니다.

저는 BLDC 모터 제어와 센서 융합 경험이 풍부합니다. ATIK에서 L-LPC, Psi-1000 같은 분석 장비를 개발하면서 FreeRTOS 기반 멀티태스킹, PID 알고리즘, Modbus 실시간 통신을 구현했습니다. 특히 회로와 펌웨어를 모두 다루기 때문에 센서 신호 처리부터 제어 알고리즘까지 end-to-end로 개발할 수 있습니다.

로봇 핸드는 처음이지만, 제가 개발했던 장비들도 정밀 제어와 다중 센서 융합이 핵심이었습니다. ADC 노이즈 필터링, 모터 제어 최적화, 실시간 통신 같은 경험을 로봇 핸드의 20-DOF 제어와 촉각 센서 융합에 적용할 수 있을 것 같습니다.

"기계는 거짓말을 안 한다"는 믿음으로 철저히 디버깅하고, 빠르게 문제를 분석해서 다각도로 해결하는 게 제 장점입니다. 원익로보틱스에서 정밀 로봇 제어 기술을 배우며 성장하고 싶습니다.

---

## 💬 예상 질문 & 답변

### 1. 로보틱스 경험이 없는데, 로봇 핸드 개발에 자신 있나요?
**답변**:
"로보틱스는 처음이지만, 제가 개발했던 분석 장비들도 센서-제어-액츄에이터가 실시간으로 연결되는 시스템이라는 점에서 본질적으로 유사합니다. 

L-LPC에서는 FreeRTOS 7개 Task로 센서 데이터 수집, FPGA 통신, Modbus 제어를 동시에 처리했고, Psi-1000에서는 PID 알고리즘으로 가스 압력을 마이크로초 단위로 제어했습니다. 이런 실시간 제어 경험이 로봇 핸드의 20-DOF 동시 제어에도 적용될 수 있을 것 같습니다.

처음 다뤄보는 기술이라도 빠르게 적응하는 편입니다. 최근 -1500V 고전압 PCB를 처음 설계했을 때도, 스파크 문제를 즉시 해결하고 고전압 설계 가이드를 학습해서 차기 보드에 반영했습니다. 로보틱스도 이런 식으로 빠르게 배워나갈 자신 있습니다."

---

### 2. ATIK에서 퇴사하시는 이유가 무엇인가요?
**답변**:
"회사 재정 악화로 연구소가 폐쇄되면서 퇴사하게 됐습니다. 4년간 분석 장비 개발로 많이 성장했지만, 이제는 새로운 분야에 도전하고 싶습니다.

로보틱스는 제게 완전히 새로운 영역이지만, 정밀 제어와 센서 융합이라는 핵심은 제가 해온 일과 맥락이 닿아있습니다. 특히 원익로보틱스의 20-DOF 로봇 핸드처럼 복잡한 시스템을 개발하면서 한 단계 더 성장하고 싶어서 지원하게 됐습니다."

---

### 3. 모터 제어 경험을 구체적으로 설명해주세요.
**답변**:
"BLDC 모터 제어와 스테퍼 모터 제어 경험이 있습니다.

**BLDC 모터**: L-LPC에서 팬 속도를 PID로 제어했고, Modbus로 외부에서 목표 속도를 받아서 실시간 조절했습니다. 홀 센서 피드백으로 RPM을 측정하고, PWM 듀티를 조절해서 정밀하게 제어했습니다.

**스테퍼 모터**: L-Titrator에서 시린지 펌프를 마이크로스텝으로 제어했습니다. 가속/감속 프로파일을 만들어서 진동을 최소화하고, pH 센서 피드백으로 적정량을 자동 조절하는 알고리즘을 구현했습니다.

로봇 핸드의 서보 모터는 경험이 없지만, PWM 제어와 엔코더 피드백이라는 기본 원리는 동일하다고 이해하고 있습니다."

---

### 4. 센서 융합 경험이 있나요?
**답변**:
"여러 센서를 동시에 처리한 경험이 많습니다.

**L-LPC**: 광학 센서(입자 검출) + 온습도 센서 + 압력 센서를 동시에 읽어서, FreeRTOS Task로 각각 처리하고 Queue로 데이터를 통합했습니다. 타이밍이 중요해서 우선순위를 조절하고 Interrupt를 활용했습니다.

**MS**: ADC 8채널 동시 샘플링으로 여러 센서 값을 읽고, DMA로 데이터를 전송해서 CPU 부하를 줄였습니다. 센서 간 타이밍 동기화와 노이즈 필터링이 중요했습니다.

로봇 핸드의 촉각 센서, 토크 센서, 엔코더 같은 멀티모달 센서도 FreeRTOS Task 구조와 DMA를 활용하면 효율적으로 처리할 수 있을 것 같습니다."

---

### 5. 실시간 제어 경험은 어느 정도인가요?
**답변**:
"마이크로초 단위 타이밍이 중요한 제어를 여러 번 해봤습니다.

**Psi-1000**: 가스 압력을 PID로 제어하는데, 샘플링 주기가 1ms였습니다. Timer Interrupt에서 ADC 값을 읽고 PID 계산 후 밸브 PWM을 업데이트하는 루프를 구현했습니다. 오버런을 방지하기 위해 계산 시간을 최적화했습니다.

**Modbus RTU**: RS-485 Half-duplex에서 Transceiver 방향 제어 타이밍이 수십 마이크로초 단위로 중요했습니다. 타이밍이 조금만 어긋나도 통신이 끊겼는데, 오실로스코프로 파형을 분석해서 정확히 맞췄습니다.

로봇 핸드의 실시간 제어도 이런 타이밍 분석과 최적화 경험이 도움이 될 것 같습니다."

---

### 6. FreeRTOS Task 우선순위를 어떻게 설계하나요?
**답변**:
"실시간성이 중요한 순서대로 우선순위를 정합니다.

**높은 우선순위**:
- 센서 데이터 수집 (놓치면 안 되는 샘플)
- 모터 제어 (정밀 타이밍 필요)

**중간 우선순위**:
- 통신 처리 (Modbus, Ethernet)
- 알고리즘 계산 (PID 등)

**낮은 우선순위**:
- UI 업데이트 (TouchGFX)
- 로그 기록

Interrupt에서는 최소한의 작업만 하고, 실제 처리는 Task에 맡깁니다. Semaphore로 Interrupt와 Task를 동기화하고, Watchdog으로 Task가 멈추지 않았는지 체크합니다."

---

### 7. 가장 어려웠던 디버깅 경험은?
**답변**:
"Psi-1000에서 I2C 통신이 간헐적으로 끊기는 문제가 있었습니다. 재현이 어려워서 2주간 매달렸는데, 오실로스코프로 파형을 계속 관찰하고 코드를 한 줄씩 검토했습니다.

결국 MCU의 I2C 기능 Errata 문서에서 버그를 발견했고, 통신이 끊겼을 때 재초기화하는 방식으로 완전히 해결했습니다. 이 경험 이후 새 MCU를 선정할 때는 Errata 문서를 먼저 확인하는 습관이 생겼습니다.

끈질기게 파고들어서 근본 원인을 찾아내는 게 제 스타일인데, 최근에는 우선순위를 정해서 효율적으로 디버깅하는 법도 배웠습니다."

---

### 8. 하드웨어와 펌웨어를 둘 다 하는 게 로봇 개발에 어떻게 도움이 될까요?
**답변**:
"로봇 핸드는 센서 신호 처리, 모터 드라이버, 통신 인터페이스 같은 하드웨어가 복잡하기 때문에, 회로를 이해하는 펌웨어 개발자가 유리합니다.

예를 들어 토크 센서 신호에 노이즈가 많다면, '펌웨어 필터링으로 해결할지, 회로에 RC 필터를 추가할지' 판단할 수 있습니다. 또 모터 드라이버 PWM 주파수와 인덕턴스의 관계를 이해하면 더 효율적인 제어가 가능합니다.

실제로 L-LPC에서 ADC 노이즈 문제를 펌웨어 필터로 임시 해결하고, 회로팀과 협업해서 PCB를 재설계한 경험이 있습니다. 이런 통합적 시각이 로봇 핸드 개발에도 큰 도움이 될 것 같습니다."

---

### 9. ROS 경험이 있나요?
**답변**:
"ROS 경험은 없습니다. 하지만 Modbus, Ethernet 같은 프로토콜로 외부 시스템과 실시간 통신한 경험이 많고, 메시지 기반 통신 구조를 이해하고 있습니다. ROS도 Topic/Service 기반 통신이라고 알고 있는데, 필요하다면 빠르게 학습해서 적용할 수 있습니다.

펌웨어 레벨에서는 센서 데이터를 수집하고 모터를 제어하는 Low-level 작업이 중요하고, ROS는 High-level 제어와 연동하는 부분이라고 이해하고 있습니다. 제 강점은 Low-level 실시간 제어이고, ROS는 입사 후 학습하겠습니다."

---

### 10. 입사 후 어떤 기여를 하고 싶으신가요?
**답변**:
"첫째, 실시간 제어 안정성을 높이고 싶습니다. 9년간 FreeRTOS로 복잡한 멀티태스킹 시스템을 개발한 경험을 로봇 핸드의 20-DOF 동시 제어에 적용하겠습니다.

둘째, 센서 노이즈 필터링과 신호 처리 노하우를 공유하고 싶습니다. ADC, 광학 센서, 압력 센서 같은 다양한 센서를 다뤄봤기 때문에, 촉각 센서나 토크 센서의 신호 품질 개선에 기여할 수 있습니다.

셋째, 하드웨어-펌웨어 통합 관점에서 디버깅 시간을 단축시키고 싶습니다. 문제가 생겼을 때 하드웨어인지 펌웨어인지 빠르게 판단해서 개발 속도를 높이겠습니다.

로보틱스는 새로운 분야지만, 제 경험을 활용하면서 정밀 로봇 제어 기술을 배우며 함께 성장하고 싶습니다."
